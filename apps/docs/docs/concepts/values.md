# Values

Values represent what is created, exchanged, and consumed in your market. They are the fundamental unit of worth.

## Types

| Type | Description | Examples |
|------|-------------|----------|
| **Product** | A tangible or digital good | Cloud Platform, Data Analytics Suite |
| **Service** | Work performed for another party | Implementation Consulting, Managed Operations |
| **Relationship** | An ongoing connection with inherent value | Strategic Partnership, Channel Alliance |
| **Right** | A permission or entitlement | Platform License, API Access Token |

## Hierarchy

Values support parent-child relationships with two relationship types:

- **Part of** &mdash; the child is a component of the parent (e.g., Compute Engine is *part of* Cloud Platform)
- **On top of** &mdash; the child is built upon or extends the parent (e.g., Analytics Suite is *on top of* Cloud Platform)

## Lifecycle Stages

Each value has a lifecycle stage: **Idea**, **Alpha**, **Beta**, **Stable**, or **Legacy**.

## Value Instances

A value instance is a concrete realization of a value &mdash; a specific license sold to a specific customer, a particular project engagement, etc. Instances track version, expiration, and the agents involved (from/to).
